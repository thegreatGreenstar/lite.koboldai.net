Usage Mode

Usage mode ?Story Mode is best for novel style writing. Adventure Mode is best for Interactive Fiction RPGs. Chat Mode is best for chat conversations with the AI. Instruct mode is for giving the AI ChatGPT styled tasks.
UI style ?Select your preferred UI style, which affects text formatting and display. Some UIs are only available for specific modes.

Chat Settings

Your Name ?The name the AI will see you as
AI Name(s) ?Name of the person(s) you want to chat with. Multiple opponents can be specified, creating a group chat, separate their names using multiple lines.
Chat pre-prompt ?Modifies the context, injecting extra tokens to improve chat quality for new chats.
Chat mode matches any name ?In Chat Mode, try to match formatting against any detected name, instead of only directly specified chat names. Does not apply to instruct mode.

Adventure Settings

Adventure pre-prompt ?Modifies the context, injecting tokens to improve adventure quality for new adventures.
Adventure roll modifer ?Adds an integer modifier to adventure mode dice rolls.

Instruct Settings

Instruct tag preset ?Quickly select between common instruct tag formats. Different models are trained with different tags.
System Tag ?The sequence to start the system prompt
Sys. Prompt ?A fixed system message sent at the very start to guide the AI behavior. Usually NOT needed
User Tag ?The sequence to start an instruction prompt
Assistant Tag ?The sequence to start an assistant response
Assist. GenTag ?The sequence used to start the final turn of an assistant response. Leave blank if unsure
Separate end tags ?Allows using separate Instruction and Response End Tags, instead of combing them with the start tag. Don't change this halfway through a story!
Assistant jailbreak ?Automatically injects a jailbreak message after every instruct query, to make the AI more likely to obey you.
Inject chat names ?Appends chat names after every instruct tag, a hybrid chat mode.

Responses

Token streaming ?Use token streaming for partial responses. SSE is recommended.
Allow continue user turns ?In instruct mode, allows the AI to continue generating the user's turn when pressing Generate More.
Allow continue AI replies ?Allow the AI to continue incomplete past responses, which can be extended by pressing submit again. Not recommended for newbies.
Allow multiline replies ?Whether to allow multiple lines in AI responses (Chat/Adventure). Disable this if the AI starts generating rubbish.
Idle responses ?Allow the AI to send more responses over time if you are idle.

Input

Send input when pressing enter ?Pressing the Enter button submits text instead of adding a newline

Formatting

Trim sentences ?Trims incomplete sentences in AI output.
Trim whitespace ?Removes trailing whitespace in AI output.
Compress newlines ?Compresses multiple newlines into one newline in AI output.
Never escape HTML ?Avoids escaping any HTML tags, allowing HTML injections. Not recommended!
Inject timestamps ?Injects timestamps into context (Chat/Instruct), allowing the AI to have a sense of time.
Fix alpaca leakage ?Prevents leaking when Alpaca instruct format is used on models trained with bad/different formats.

Appearance Settings

UI style ?Select your preferred UI style, which affects text formatting and display. Some UIs are only available for specific modes.
Color scheme ?Set UI color scheme. Does not apply to corpo mode.
Enable markdown ?Allows the UI to use markdown formatting such as quotes and code blocks.
Render markdown while streaming ?Attempt to render markdown when streaming. May result in wonky output or lag on older devices.
Enable LaTeX ?Allows the UI to render LaTeX within markdown formatting (Needs Markdown).
Show Nametags ?Controls if user and AI nametags are shown or hidden. Recommended.
Autoscroll ?Automatically scrolls the text window down when new text is generated
Viewport width ?Controls horizontal scaling of the viewport window
Unlock scroll height ?Unlocks the text viewport, allowing for infinite height without scrolling
Max Displayed Text Characters ?If set above 0, will limit the number of text characters rendered to display (any earlier text will still be submitted but stay hidden).
Background image ?Set an image as a decorative background. Stored locally.
Inverted colors ?Inverts all colors, simple light mode
Side panel mode ?Displays the settings and context panels permanently on the sides, instead of as popups. Not available on Mobile displays.
Word frequency analysis ?Shows a tool that displays word frequency and locations, inspired by trincadev MyGhostWriter. Counts across newlines may be inaccurate.
Custom CSS ?Allows you to load third-party CSS styles (caution).
Toggle Fullscreen ?Attempts to switch to fullscreen mode. This setting cannot be saved.

Token Settings

Stop Sequences ?Allows text generation to stop early if any of the specified sequences are hit. Using default stops adds extra sequences for instruct tags and character names.
Stop Sequences may be unavailable.
Include Default Stop Sequences ?Adds a few default stop sequences from instruct tags and chat names. Leave enabled if unsure.
Extra Stopping Sequences ?If you want multiple sequences, separate them with the following delimiter: ||$||
Logit Biases ?Specify a dictionary of token IDs to modify the probability of occuring. Enter OpenAI-formatted logit bias dictionary. Each key is the integer token IDs and their values are the biases (-100.0 to 100.0). Leave blank to disable.
Logit bias may be unavailable.
Input is a JSON object, reference here.
Input Strings Instead of IDs (Uses KCPP Tokenizer) ?If enabled, allows you to input strings instead of only token IDs, and tokenizes them for you.
Phrase / Word Ban (Anti-Slop) ?Prevents specific words or phrases from being generated, either modifying model vocab or by backtracking and regenerating when they appear. If you want multiple sequences, separate them with the following delimiter: ||$||
Note: If you're trying to ban a specific token by ID, you should use Logit Bias instead!
Phrase banning may be unavailable.
Regex Replace ?Allows transforming incoming text with regex patterns, modifying all matches. Replacements will be applied in sequence.
Thinking / Reasoning Tags ?Allows hiding or removing thinking tags. Specifies how to handle output from reasoning models, to hide, remove, or ignore the Chain-Of-Thought.
Start Think Tag ?Think tag to match for start of CoT.
Stop Think Tag ?Think tag to match for end of CoT.
CoT Display ?Controls the visibility of chain-of-thought thinking sections in the UI. Does not affect submitted thinking tags.
CoT Submitted ?Controls which thinking tags are submitted to the AI.
Insert Thinking ?Controls whether thinking tags are inserted or not. To Prevent thinking, set Non-Thinking instruct tags instead!
Hide Streaming Thinking ? Hides incomplete thoughts while streaming is in progress.
Handle Mismatched Thinking ? Handles cases where the closing think block exists without the opening. Note that this only visually hides the broken thinking, it will still be treated like normal context.
Reasoning Effort ?If set, restricts thinking length to this many tokens before forcing a reply.
Placeholder Tags ?Configure automatic substitutions for placeholders in text.
Enable Placeholder Tags ? If enabled, substitutes placeholders like {{user}} and other custom placeholders here that get swapped on submit.
Randomness Seed ?A seed to generate randomness for {{pick}}, {{roll}} and {{random}}. Set to -1 to disable these three placeholders.
Classifier-Free Guidance ?Functions as a negative prompt when Guidance Scale is above 1, and a positive prompt at Guidance Scale is below 1. Disabled if scale is exactly 1 or CFG prompt is blank.
Classifier-Free Guidance may be unavailable.
